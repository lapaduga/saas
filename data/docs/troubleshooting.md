# Решение проблем CloudApp

## Частые проблемы и решения

### Проблемы с авторизацией

#### Не могу войти в аккаунт

**Причины и решения:**

1. **Неверный пароль**
   - Используйте «Забыли пароль?» на странице входа
   - Проверьте раскладку клавиатуры (Caps Lock)
   - Убедитесь, что нет лишних пробелов

2. **Аккаунт заблокирован**
   - После 5 неудачных попыток входа аккаунт блокируется на 15 минут
   - Подождите или напишите в поддержку для немедленной разблокировки

3. **Email не подтверждён**
   - Проверьте папку «Спам» на наличие письма подтверждения
   - Запросите повторное подтверждение на странице входа

4. **Проблемы с SSO**
   - Убедитесь, что SSO-сессия активна в корпоративном приложении
   - Очистите кэш браузера и cookies
   - Попробуйте режим инкогнито
   - Проверьте, что время на сервере и клиенте синхронизировано

#### 403 Forbidden при доступе к файлу

**Причины и решения:**

1. **Нет прав доступа**
   - Обратитесь к владельцу файла для предоставления доступа
   - Проверьте, что ваша роль позволяет данный тип операции

2. **Истёк срок действия шаринга**
   - Попросите владельца продлить ссылку или создать новую

3. **SSO-сессия истекла**
   - Выйдите и войдите заново через SSO
   - Попросите администратора проверить настройки SSO

4. **Файл удалён создателем**
   - Проверьте корзину (если у вас есть доступ)
   - Свяжитесь с создателем файла

#### 401 Unauthorized

**Решения:**

- Проверьте API-ключ: `GET /v1/users/me`
- Убедитесь, что ключ не отозван
- Проверьте заголовок: `Authorization: Bearer sk_live_...`
- Для OAuth 2.0: обновите access token через refresh token

### Проблемы с загрузкой файлов

#### Файл не загружается

**Диагностика:**

1. **Проверьте размер файла**
   - Free: макс. 500 MB
   - Basic: макс. 2 GB
   - Pro: макс. 10 GB
   - Enterprise: макс. 100 GB

2. **Проверьте формат**
   - CloudApp поддерживает все распространённые форматы
   - Исключение: исполняемые файлы (.exe, .bat, .sh) на Free-плане

3. **Проверьте соединение**
   - Минимальная скорость: 1 Mbit/s
   - Используйте проводное соединение вместо Wi-Fi
   - Отключите VPN если он замедляет соединение

4. **Ошибки браузера**
   - Очистите кэш и cookies
   - Отключите расширения браузера
   - Попробуйте другой браузер (Chrome, Firefox, Edge)

**Код ошибки 413 (Payload Too Large):**

Файл превышает лимит тарифа. Повысьте тариф или разбейте файл.

**Код ошибки 500 при загрузке >100MB:**

Известная проблема с таймаутом. Используйте API для загрузки больших файлов:

```bash
curl -X POST https://api.cloudapp.com/v1/files \
  -H "Authorization: Bearer YOUR_KEY" \
  -F "file=@large-file.zip"
```

#### Загрузка зависает на 99%

**Решения:**

1. Подождите 5 минут — файл может обрабатываться на сервере
2. Проверьте статус: `GET /v1/files/:id`
3. Если статус не обновляется — отмените и повторите загрузку
4. Для файлов >1GB используйте API с retry-логикой

### Проблемы с API

#### Ошибка 429 Too Many Requests

**Решения:**

1. Подождите до истечения окна rate limit (см. `Retry-After`)
2. Реализуйте exponential backoff:

```javascript
async function fetchWithRetry(url, options, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    const response = await fetch(url, options);
    if (response.status === 429) {
      const retryAfter = response.headers.get('Retry-After') || Math.pow(2, i);
      await new Promise(r => setTimeout(r, retryAfter * 1000));
      continue;
    }
    return response;
  }
  throw new Error('Max retries exceeded');
}
```

3. Используйте batch-запросы: `POST /v1/files/batch`
4. Кэшируйте результаты GET-запросов
5. Повысьте тариф для увеличения лимитов

#### Ошибка 400 Bad Request

**Частые причины:**

- Невалидный JSON в теле запроса
- Обязательные поля не заполнены
- Невалидный формат данных (email, URL, дата)

**Пример валидации:**

```json
{
  "error": {
    "code": "validation_error",
    "message": "Invalid email format",
    "details": {
      "field": "email",
      "value": "not-an-email"
    }
  }
}
```

#### Ошибка 500 Internal Server Error

**Действия:**

1. Проверьте статус системы: `status.cloudapp.com`
2. Повторите запрос через 30 секунд
3. Если ошибка повторяется — напишите в поддержку с:
   - Time-stamp запроса
   - Request ID (из заголовка `X-Request-Id`)
   - Тело запроса (без секретов)

### Проблемы с вебхуками

#### Вебхук не доставляется

**Диагностика:**

1. Проверьте URL: он должен отвечать 200 OK
2. Проверьте, что сервер доступен из интернета
3. Проверьте логи вашего сервера
4. Убедитесь, что URL использует HTTPS

**Настройка тестового эндпоинта:**

```bash
# Используйте webhook.site для тестов
curl -X POST https://api.cloudapp.com/v1/webhooks \
  -H "Authorization: Bearer YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://webhook.site/YOUR-ID", "events": ["file.created"]}'
```

#### Вебхук доставлен, но сигнатура не проходит верификацию

**Решения:**

1. Убедитесь, что используете правильный секрет
2. Проверьте, что тело запроса не модифицировано промежуточными серверами
3. Используйте `crypto.timingSafeEqual` вместо `===` для сравнения

### Проблемы с производительностью

#### Медленная загрузка файлов

**Оптимизации:**

1. Используйте CDN-URL для скачивания: `https://storage.cloudapp.com/...`
2. Разбивайте большие файлы на части
3. Используйте parallel upload (до 5 частей одновременно)
4. Проверьте скорость вашего подключения

#### Медленный поиск

**Оптимизации:**

1. Уточните поисковый запрос (меньше слов → быстрее)
2. Используйте фильтры (path, type) для сужения области
3. Кэшируйте результаты поиска на клиенте

### Проблемы с интеграциями

#### SSO: SAML assertion invalid

**Решения:**

1. Убедитесь, что часы на сервере Okta/Azure синхронизированы (NTP)
2. Проверьте маппинг атрибутов
3. Убедитесь, что ACS URL правильный
4. Проверьте сертификат IdP

#### 1С:ERP дублирует записи

**Решения:**

1. Добавьте заголовок `X-Idempotency-Key` в каждый запрос
2. Используйте уникальный ID для каждой операции
3. Проверьте логи webhook-обработчика на стороне CloudApp

## Диагностика

### Проверка статуса системы

```bash
# Health check
curl https://api.cloudapp.com/v1/health

# Детальный статус
curl https://status.cloudapp.com/api/v2/summary.json
```

### Проверка API-ключа

```bash
curl -H "Authorization: Bearer YOUR_KEY" \
  https://api.cloudapp.com/v1/users/me
```

Обратите внимание на заголовки:

```

X-RateLimit-Remaining: 497
X-RateLimit-Reset: 1721155260
```

## Получение помощи

### Самостоятельная диагностика

1. Проверьте FAQ: `support.cloudapp.com/faq`
2. Проверьте статус системы: `status.cloudapp.com`
3. Поищите в документации: `docs.cloudapp.com`
4. Используйте AI-ассистента в интерфейсе

### Обращение в поддержку

При обращении предоставьте:

1. **Email аккаунта**
2. **Описание проблемы** (что произошло, что ожидали)
3. **Шаги для воспроизведения**
4. **Коды ошибок** (если есть)
5. **Скриншоты** (если применимо)
6. **ID тикета** (если это follow-up)

**Каналы поддержки:**

- **Email:** support@cloudapp.com
- **Чат:** В правом нижнем углу интерфейса
- **Телефон:** +7 (800) 555-35-35 (Pro/Enterprise)
- **Срочная поддержка:** urgent@cloudapp.com (Enterprise)
